## Backup / export / reconstruction

- **D1 export**: `wrangler d1 execute high-signal-db --remote --command "SELECT * FROM <table>" > backup.sql` per table. Operator-run, not automated (proposal non-goal: no production mutation).
- **Git is the source of truth** for `signals/`, `data/equities-snapshot.jsonl`, `data/product-flow-refresh.jsonl`, `apps/web/src/data/*.json`, `workers/api/src/lib/label-backtest.json`. Cloning the repo reconstructs these.
- **D1 audit tables** (`ingest_runs`, `llm_runs`, `score_runs`) are not reconstructable. They are the debug window. Acceptable loss: 30 days. Operator may export periodically.
- **User state** (`delivery_*`, `watchlists_*`) is not reconstructable. Operator must back up D1 before any destructive action.

## Migration guards

- Migrations live in `packages/db/migrations/`; `meta/` tracks applied state.
- Remote apply is a manual operator step: `pnpm db:migrate:remote`.
- No migration is auto-applied by Foundry or any cron. The coverage audit reports
  pending migrations (files in `migrations/` not in `meta/`) as a blocker for
  any job that depends on the new table.
- The `automate-high-signal` proposal explicitly forbids data migration as part
  of this change. This registry documents the *current* state only.

## Provenance

- Every `signals/*.md` carries cited URLs (≥ 2 sources required — cite-or-kill).
- `events.raw_hash` + `source_documents.raw_hash` tie every raw observation to
  its fetched source.
- `evidence` rows link signals to events to source URLs.
- `cited_url_index` is the denormalized lookup.
- `llm_runs` records `signal_slug`, `model`, `prompt_version`, `tokens_in/out`,
  `latency_ms`, `accepted`, `reason` — **but not the raw prompt body** (sanitized
  to `request_json`/`response_json` shapes; the audit adapter redacts
  credential-shaped values before any fleet report).

## Restore verification

- **Reconstructable from git**: clone fresh → `pnpm install` → `pnpm db:seed:remote` → `pnpm signals:sync:remote --force` → `pnpm daily:snapshot` → `pnpm market:snapshot` → `pnpm equities:snapshot` → `pnpm tickers:bundle` → `pnpm d2c:bundle`. Verify with `pnpm source:diagnose` and the `/explore` sitemap.
- **Reconstructable from upstream APIs**: re-run the relevant cron workflow (`workflow_dispatch`) for the affected date window. Idempotency guards (`raw_hash`, hour-bucket ids, `(ticker,date)` uniques, `(niche,snapshot_date)` uniques) prevent duplicates.
- **Not reconstructable (audit/user state)**: restore from operator D1 export. No automated restore path — recorded as an accepted exception per the spec's data-durability requirement.
